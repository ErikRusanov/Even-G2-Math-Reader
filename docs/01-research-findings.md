# Even Realities G2 — Research Findings (Math Teleprompter)

> Generated by a multi-agent deep-research workflow (6 search angles → 25 sources →
> 123 extracted claims → 3-vote adversarial verification → 10 confirmed).
> Date: 2026-06-20. Confidence levels and sources are explicit per finding.
> See [`04-sources.md`](./04-sources.md) for the full citation list.

## TL;DR — Is this buildable?

**Yes.** The Even Realities G2 has an **official developer platform (EvenHub)**. Apps are
ordinary **web apps (HTML/CSS/TypeScript)** that run inside the **Even Hub companion phone
app's WebView**. The glasses themselves run no app logic — they are a **576×288 px
monochrome-green display + input device** (touchpad, gestures, voice, R1 ring).

The **hardest constraint is rendering dense math**. The native text path enforces ~25
characters per line — LaTeX-grade fractions, sums, matrices, and subscripts are **impossible
as text**. The only viable path is to **pre-render each formula (or whole page) to a 4-bit
grayscale bitmap** with KaTeX/MathJax and push it as an **image payload**. The exact image
API limits are the #1 thing to validate on real hardware (see open questions).

## 2. Deployment & runtime model — `confidence: HIGH`

> *"Your code lives on the phone. The glasses render and capture input."* — official docs

- Apps are **web apps glued to the glasses by the Even Hub SDK**, executed in the **phone
  app's WebView**. The glasses are **display + input only**.
- Deployment: **QR sideloading via `evenhub-cli`** after authentication.
- Confirmed by official docs + community projects (`even-toolkit`, `g2_helloworld`, `even-dev`).

## 3. Display hardware & limits — `confidence: HIGH (specs) / MEDIUM (exact pixel limits)`

- **576×288 px, monochrome green, 4-bit grayscale (16 intensity levels).** No RGB.
- **~1200 nits Micro-LED** (notebookcheck).
- **Native teleprompter mode: exactly 10 lines/page, ~25 chars/line, ~7 lines visible at once.**
- **Image containers reported limited to ~200×100 px; animation frames ~30×30 px** (Zenn SDK
  article — *blog quality*). **Treat as a hypothesis until verified** — a refuted competing
  claim suggested 576×136 1-bit BMP via the older EvenDemoApp path.

> ⚠️ Tension to resolve on hardware: is the usable image surface **200×100** (SDK container)
> or near **full-width 576**? This directly determines how much math fits per page.

## 4. BLE protocol — `confidence: HIGH for G1, MEDIUM for G2`

Relevant only if you bypass the official SDK and talk to the glasses directly (not recommended
for v1, but useful to understand the substrate):

- **Nordic UART Service (NUS) UUIDs** (confirmed across multiple primary sources):
  - Service: `6E400001-B5A3-F393-E0A9-E50E24DCCA9E`
  - Write (TX): `6E400002-B5A3-F393-E0A9-E50E24DCCA9E`
  - Read/Notify (RX): `6E400003-B5A3-F393-E0A9-E50E24DCCA9E`
- **G1 uses two BLE radios — one per arm** — each a separate UART stream; sends are sequential
  (left, then right).
- **G2 teleprompter pages** transmit as **10-line UTF-8 chunks with `\n` separators**, page
  numbers as **0-indexed varints** (binary-level analysis in `i-soxi/even-g2-protocol`).
- ⚠️ All G2 BLE detail is **reverse-engineered**, not official, and may not match current
  firmware. Payload/MTU specifics (180/194/204-byte packet claims) were **refuted/unconfirmed** —
  do not rely on them.

## 5. Autoscroll / scroll speed — `confidence: MEDIUM`

The **native** Even teleprompter offers **three scroll modes** (primary source,
evenrealities.com/teleprompter-glasses):

1. **AI-paced** — adapts to the speaker's speech rate via voice recognition.
2. **Fixed speed** — a consistent WPM setting.
3. **Manual** — via the **Even R1 smart ring** or **tapping the G2 TouchPad**.

> ⚠️ These are **native first-party features**. Whether a **custom SDK web app** can hook the
> same TouchPad / R1 input events is **unconfirmed** — this is a key open question for our
> on-glasses speed control. Fallback: drive autoscroll from a `setInterval` in the WebView and
> set speed from the companion phone UI.

## 6. Math rendering — `confidence: MEDIUM (engineering inference)`

- **No existing G2 math/LaTeX project was found.** This is novel territory.
- LaTeX-grade math **cannot** be rendered as text (25 char/line limit).
- **Recommended pipeline:** LaTeX → **KaTeX** (fast) or MathJax → SVG/canvas → downscale to
  **4-bit grayscale** → send as image. Use **Floyd–Steinberg dithering** and **invert to
  white-on-black** (KaTeX renders dark-on-transparent) to preserve thin strokes (fraction bars,
  Σ, Greek) at low bit depth.
- Mature LaTeX→image references: `DMOJ/texoid`, `mneri/pnglatex`, `klatexformula`.

## Open questions — VALIDATE THESE FIRST (block the design)

1. **Input access:** Does the EvenHub SDK expose **TouchPad tap / R1 ring** events to
   third-party web apps? (Determines on-glasses speed control.)
2. **Image API:** What exactly does `even_hub_sdk` accept for images — grayscale PNG up to
   200×100? Does sending an image **bypass** the 10-line teleprompter page structure or must it
   be wrapped in page packets?
3. **Payload/MTU:** A 200×100 4-bit image ≈ 10 KB. Does the SDK **chunk transparently** or must
   we fragment manually?
4. **Full-surface mode:** Can a custom app use the **full 576×288** as a canvas, or is content
   always constrained to the narrow teleprompter column?

## Caveats

- G2 BLE/protocol facts are reverse-engineered, not official, and firmware-version-sensitive.
- The math-via-image pipeline is an **inference from confirmed hardware constraints** — no
  existing project validates it empirically. Build a hardware spike early.
- SDK v0.0.10 is early; image-size limits and display-mode APIs may change.
- Several precise numbers (exact packet sizes, 576×136 BMP, 2000-char text path) were
  **refuted or split** in verification — see [`04-sources.md`](./04-sources.md) refuted list.
